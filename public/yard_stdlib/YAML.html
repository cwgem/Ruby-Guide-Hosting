Overview

Hack to handle syck's DefaultKey bug with psych

Quick note! If/when psych loads in 1.9, it will redefine YAML to point to Psych by removing the YAML constant. Thusly, over in Gem.load_yaml, we define DefaultKey again after proper yaml library has been loaded.

All this is so that there is always a YAML::Syck::DefaultKey class no matter if the full yaml library has loaded or not.
